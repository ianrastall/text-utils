```markdown
   1: # Case Converter Tool: Full Technical Walkthrough
   2: 
   3: This document explains how the Case Converter page works end to end, including UI structure, runtime flow, conversion algorithms, shared dependencies, and integration with the wider site.
   4: 
   5: ## Scope
   6: 
   7: The active implementation of the tool is in:
   8: 
   9: - `case-converter.html`
  10: 
  11: Shared code and integration points that directly affect behavior:
  12: 
  13: - `js/theme.js`
  14: - `css/style.css`
  15: - `index.html`
  16: - `sitemap.xml`
  17: 
  18: ## What the Tool Does
  19: 
  20: The tool reads text from an input textarea, converts it to one of nine casing styles, and writes the result to an output textarea in real time:
  21: 
  22: - `UPPERCASE`
  23: - `lowercase`
  24: - `Title Case`
  25: - `Sentence case`
  26: - `camelCase`
  27: - `PascalCase`
  28: - `snake_case`
  29: - `kebab-case`
  30: - `InVeRt CaSe`
  31: 
  32: ## Page Composition (`case-converter.html`)
  33: 
  34: The page is standalone. It has:
  35: 
  36: - Global shell: header, logo, controls, footer
  37: - Tool section: title, action buttons, options panel, input/output panes, status bar
  38: - Inline script that contains all case-conversion logic and event binding
  39: 
  40: ### Structural Elements and IDs
  41: 
  42: Main interactive nodes:
  43: 
  44: - `#accentColor`: accent color dropdown (driven by `js/theme.js`)
  45: - `#themeToggle`: light/dark toggle button (driven by `js/theme.js`)
  46: - `#backButton`: navigates back to `index.html`
  47: - `#clearAll`: clears input/output
  48: - `#copyOutput`: copies output text via Clipboard API
  49: - `#caseType`: selects conversion mode
  50: - `#inputText`: source textarea
  51: - `#outputText`: readonly result textarea
  52: - `#inputStats`: input chars/lines label
  53: - `#outputStats`: output chars/lines label
  54: - `#statusMessage`: short status text (`Ready`, `Success`, etc.)
  55: - `#toolStats`: operation timing text
  56: 
  57: ## Runtime Lifecycle
  58: 
  59: ### 1. Browser Loads Assets
  60: 
  61: - Material Icons stylesheet is loaded from Google Fonts
  62: - `css/style.css` is loaded for layout and theme tokens
  63: - `js/theme.js` is loaded before the tool's inline script
  64: 
  65: ### 2. `js/theme.js` Initializes Shared Theme Features
  66: 
  67: `js/theme.js` runs two immediately invoked function expressions:
  68: 
  69: - Theme/accent manager:
  70:   - Reads `localStorage.theme` (default `dark`)
  71:   - Reads `localStorage.accentColor` (default `#0d9488`)
  72:   - Applies `data-theme` on `<html>`
  73:   - Applies CSS variables `--accent` and `--accent-light`
  74:   - Wires `#themeToggle` click and `#accentColor` change
  75: - Seasonal snow manager:
  76:   - Active only during holiday window (post-Thanksgiving through Jan 1)
  77:   - Injects a `#snowToggle` button into `.controls`
  78:   - Persists state in `localStorage.holiday_snow_enabled`
  79:   - Renders animated snow on a fixed fullscreen canvas
  80: 
  81: Because Case Converter has the expected header structure (`.controls` plus theme controls), both shared behaviors attach automatically.
  82: 
  83: ### 3. Inline Tool Script Binds Case Converter Logic
  84: 
  85: The inline script in `case-converter.html`:
  86: 
  87: - Caches DOM references by ID
  88: - Declares helper functions
  89: - Registers event listeners
  90: - Calls `updateStats()` once for initial render
  91: 
  92: No framework is used. All state is managed directly in DOM values and labels.
  93: 
  94: ## Internal State Model
  95: 
  96: There is no central state object. Effective state is stored in:
  97: 
  98: - Form controls:
  99:   - `inputText.value`
 100:   - `caseTypeSelect.value`
 101:   - `outputText.value`
 102: - UI labels:
 103:   - `statusMessage.textContent` and class (`success` or `error`)
 104:   - `toolStats.textContent`
 105:   - stats labels (`inputStats`, `outputStats`)
 106: 
 107: Persisted cross-page state comes only from `js/theme.js`:
 108: 
 109: - `localStorage.theme`
 110: - `localStorage.accentColor`
 111: - `localStorage.holiday_snow_enabled` (seasonal only)
 112: 
 113: The Case Converter logic itself does not persist user text or selected case mode.
 114: 
 115: ## Conversion Pipeline
 116: 
 117: Main function:
 118: 
 119: - `convertCase()`
 120: 
 121: Execution flow:
 122: 
 123: 1. Read `inputText.value` and selected case from `#caseType`.
 124: 2. If input is empty:
 125:    - clear output
 126:    - call `updateStats()`
 127:    - return
 128: 3. Start timer with `performance.now()`.
 129: 4. Dispatch to converter function via `switch(caseType)`.
 130: 5. Write result to output.
 131: 6. Set status to success and write processing time in milliseconds.
 132: 7. On exception:
 133:    - write `Error: ...` to output
 134:    - set error status
 135:    - set `toolStats` to `Processing failed`
 136: 8. Call `updateStats()`.
 137: 
 138: ## Converter Functions and Behavior Details
 139: 
 140: ### `toUpperCase(text)`
 141: 
 142: - Uses `text.toUpperCase()`.
 143: - Locale-insensitive default JavaScript case mapping.
 144: 
 145: ### `toLowerCase(text)`
 146: 
 147: - Uses `text.toLowerCase()`.
 148: - Locale-insensitive default JavaScript case mapping.
 149: 
 150: ### `toTitleCase(text)`
 151: 
 152: - Uses regex: `text.replace(/\b\w/g, char => char.toUpperCase())`.
 153: - Uppercases only characters at word boundaries.
 154: - Does not lowercase remaining letters in each word.
 155: 
 156: Example:
 157: 
 158: - Input: `hELLo wORLD`
 159: - Output: `HELLo WORLD`
 160: 
 161: ### `toSentenceCase(text)`
 162: 
 163: - Uses: `text.charAt(0).toUpperCase() + text.slice(1).toLowerCase()`.
 164: - Only first character of full string is uppercased.
 165: - Everything after first character is lowercased.
 166: - Does not detect multiple sentences.
 167: 
 168: ### `toCamelCase(text)`
 169: 
 170: Pipeline:
 171: 
 172: 1. Regex transform: `/(?:^\w|[A-Z]|\b\w)/g`
 173:    - first match lowercased
 174:    - subsequent matches uppercased
 175: 2. Remove whitespace: `.replace(/\s+/g, '')`
 176: 3. Remove non-alphanumeric chars: `.replace(/[^a-zA-Z0-9]/g, '')`
 177: 
 178: Effects:
 179: 
 180: - Drops punctuation/symbols.
 181: - Keeps digits.
 182: - Can collapse separators aggressively.
 183: 
 184: ### `toPascalCase(text)`
 185: 
 186: Pipeline:
 187: 
 188: 1. Regex transform: `/(?:^\w|[A-Z]|\b\w)/g` -> uppercase all matches
 189: 2. Remove whitespace
 190: 3. Remove non-alphanumeric chars
 191: 
 192: Result starts with uppercase alphanumeric segment.
 193: 
 194: ### `toSnakeCase(text)`
 195: 
 196: Pipeline:
 197: 
 198: 1. Spaces to underscores: `.replace(/\s+/g, '_')`
 199: 2. Split lower->upper transitions: `.replace(/([a-z])([A-Z])/g, '$1_$2')`
 200: 3. Lowercase all
 201: 4. Remove disallowed chars: `.replace(/[^a-z0-9_]/g, '')`
 202: 
 203: Effects:
 204: 
 205: - Existing punctuation is removed (except underscore).
 206: - Handles `camelCase` into `camel_case`.
 207: 
 208: ### `toKebabCase(text)`
 209: 
 210: Same as snake-case logic but with hyphens:
 211: 
 212: 1. Spaces to `-`
 213: 2. Split lower->upper transitions with `-`
 214: 3. Lowercase all
 215: 4. Remove disallowed chars: `.replace(/[^a-z0-9-]/g, '')`
 216: 
 217: ### `invertCase(text)`
 218: 
 219: - Splits string into chars.
 220: - For each char:
 221:   - if `char === char.toUpperCase()`, use lowercase
 222:   - else use uppercase
 223: - Joins chars back.
 224: 
 225: Notes:
 226: 
 227: - Non-letters (digits, punctuation) usually remain unchanged because uppercase/lowercase forms are identical.
 228: 
 229: ## Event Wiring and Interaction Model
 230: 
 231: ### Real-time conversion on typing
 232: 
 233: - `input` event on `#inputText`:
 234:   - `updateStats()`
 235:   - `convertCase()`
 236: 
 237: This means conversion happens on every keystroke with no debounce.
 238: 
 239: ### Re-convert on mode change
 240: 
 241: - `change` event on `#caseType` calls `convertCase()`.
 242: 
 243: ### Copy output
 244: 
 245: - Click on `#copyOutput`:
 246:   - No-op if output is empty
 247:   - Uses `navigator.clipboard.writeText(outputText.value)`
 248:   - On success:
 249:     - status message: `Copied to clipboard!`
 250:     - class: `success`
 251:     - resets to `Ready` after 2 seconds
 252:   - On failure:
 253:     - status message: `Failed to copy`
 254:     - class: `error`
 255: 
 256: ### Clear fields
 257: 
 258: - Click on `#clearAll`:
 259:   - clears both textareas
 260:   - updates stats
 261:   - status message `Cleared`
 262:   - resets to `Ready` after 1.5 seconds
 263: 
 264: ### Back navigation
 265: 
 266: - Click on `#backButton`:
 267:   - `window.location.href = 'index.html'`
 268: 
 269: ### Keyboard shortcut
 270: 
 271: - Document-level `keydown` listener:
 272:   - if Ctrl/Cmd + Enter, prevent default and run `convertCase()`
 273: 
 274: ## Status and Metrics Behavior
 275: 
 276: `#statusMessage` states used by script:
 277: 
 278: - `Ready`
 279: - `Success`
 280: - `Copied to clipboard!`
 281: - `Failed to copy`
 282: - `Cleared`
 283: - `Error`
 284: 
 285: `#toolStats` content:
 286: 
 287: - default HTML text: `No operations yet`
 288: - on successful conversion: `Processed in Xms`
 289: - on conversion failure: `Processing failed`
 290: 
 291: Timing is measured only around conversion logic (`switch` and function call), not full event dispatch or rendering.
 292: 
 293: ## Styling and Layout (`css/style.css`)
 294: 
 295: Case Converter relies on global utility/layout classes:
 296: 
 297: - Theme variables via `:root` and `[data-theme="light"]`
 298: - Header and control styling (`header`, `.controls`, `.accent-selector`)
 299: - Tool layout (`.tool-header`, `.tool-actions`, `.options-panel`, `.io-container`)
 300: - Input/output styles (`textarea`, `.output`, `.stats`)
 301: - Status styles (`.success`, `.warning`, `.error`)
 302: - Responsive behavior:
 303:   - `.io-container` collapses to single column at `max-width: 768px`
 304:   - `.tool-header` stacks vertically on small screens
 305: 
 306: ## How the Tool Interoperates with the Rest of the Site
 307: 
 308: ### Dashboard entry (`index.html`)
 309: 
 310: - Tool metadata array includes:
 311:   - `id: 'case-converter'`
 312:   - name/icon/category/description
 313: - Card links use pattern `${tool.id}.html`, which resolves to `case-converter.html`.
 314: 
 315: So the tool is discovered and opened through the homepage grid.
 316: 
 317: ### Search engine/site indexing (`sitemap.xml`)
 318: 
 319: - Includes URL:
 320:   - `https://text-utils.net/case-converter.html`
 321: 
 322: ### Shared header behavior (`js/theme.js`)
 323: 
 324: - Theme and accent controls are common across tools.
 325: - Seasonal snow toggle is injected into Case Converter header controls during holiday season.
 326: 
 327: ## Important Edge Cases and Current Limitations
 328: 
 329: - Empty text line counts:
 330:   - `updateStats()` uses `split('\n').length`, so empty string counts as 1 line.
 331: - Title case is not strict title normalization:
 332:   - it uppercases boundary letters but does not lowercase the rest.
 333: - Sentence case only treats the whole text as one sentence.
 334: - No locale-aware case transform options (for special language rules).
 335: - No debounce/throttling for very large pasted input.

 338:
```
